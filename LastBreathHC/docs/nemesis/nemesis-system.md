# Nemesis System Documentation

This document explains how the LastBreathHC nemesis system works, what players/admins can do with it, and how **tokens** interact with the system.

## 1) What the Nemesis system does

The nemesis system turns dangerous hostile mobs into persistent "captains" with identity, progression, traits, names, and lifecycle state.

At a high level it provides:
- Persistent nemesis captain records (`CaptainRecord`) stored in the captain registry/serializer.
- Spawn orchestration around player activity and random overworld promotions.
- Captain combat behavior, progression XP, trait evolution, and rivalry interactions.
- UI feedback (boss bar + action bar "Nemesis nearby").
- Kill rewards from captains, including a **Nemesis Token** item drop.

## 2) Captain lifecycle

Captains move through states managed by `CaptainStateMachine`:
- `DORMANT` (created / idle record)
- `ACTIVE` (spawned and hunting)
- `COOLDOWN` (escaped/despawned; waiting before next eligible spawn)
- `DEAD` (killed)
- `RETIRED` (manually retired/admin)

Spawns are coordinated by `CaptainSpawner` using player movement/join/death interest signals, per-world attempt caps, world caps, and random director ticks.

## 3) Naming behavior (current)

Captain names are generated by `CaptainNameGenerator` and include:
- Prefix (optional, from matching modifier)
- Orcish first + surname/suffix
- **Exactly one descriptor** chosen from eligible descriptor candidates (trait suffix and/or modifier descriptor fields), with weighted epithet fallback

This keeps names unique but concise while avoiding stacked descriptors.

## 4) Player-facing signals and encounters

Players are informed by:
- Broadcasts when a captain rises.
- Nemesis taunt messages.
- Boss bar + action bar updates when an active captain is nearby.

Admin/player controls are under `/nemesis` (`list`, `nearby`, `info`, `hunt`, admin spawn/retire/clear/debug variants).

## 5) Rewards and token behavior

### Nemesis Token (from nemesis captain kills)
- Source: dropped by `NemesisRewardService` on captain death (killer required).
- Item form: a `NETHER_STAR` named `Nemesis Token`.
- Cooldown: per-player reward cooldown (`nemesis.rewards.playerCooldownMs`) throttles repeated reward grants.
- Bonus reward: chance for an extra diamond (`nemesis.rewards.bonusChance`).

> Current status: the codebase currently defines Nemesis Token generation/drop, but does **not** yet define a dedicated Nemesis Token sink/consumption workflow in the same package.

### Revival Token (`ReviveToken`)
- Purpose: death protection / softening final-death outcomes.
- Used by:
  - Hardcore death interception in `DeathListener` (consumed to prevent lethal outcome).
  - Nemesis death outcome resolver (`TokenAwareDeathOutcomeResolver`) to classify player death as soft vs final.
  - GUI revive confirmation flow (`ReviveGUI`) when confirming token use.
  - Head interaction flow that also checks/consumes revive token via helper.
- Storage behavior: `ReviveTokenHelper` checks inventory, off-hand, and nested shulker containers.

### Revival Name Token (`ReviveGuiToken`)
- Purpose: revive a banned player name through chat-driven revive prompt.
- Use flow: right-click with token -> input target name in chat -> token consumed on successful pardon/revive.
- Main logic is in `ReviveNameGUI`.

## 6) Important config knobs for nemesis spawning

Primary configuration section: `nemesis.spawn` in `config.yml`.

Key controls:
- `director.minTickDelay` / `maxTickDelay`: how often director ticks run.
- `interest.movementChance`, `interest.joinChance`, `interest.deathChance`: how likely player events queue hunt interest.
- `world.activeCaptainCap`: max active captains in world.
- `world.perMinuteAttemptCap`: hard cap on spawn attempts per minute per world.
- `overworldRandomCaptain.chancePerDirectorTick`: random promotion chance per director tick.
- `recordCooldownMs`: cooldown applied after escape/despawn before captain can re-enter rotation.

## 7) Operations notes for admins

- Use `/nemesis nearby` for a GUI view of nearby captains.
- Use `/nemesis info <id|name>` to inspect level/traits.
- Use `/nemesis debug` and subcommands for state inspection and killer attribution diagnostics.
- Use `/nemesis clear confirm` cautiously: it clears all captain records and despawns minions.
